12 STRIDE Threat Analysis

STRIDE Threat Analysis

S Spoofing C2 server returns fake Apache 404 page (Cyrillic charset). Smoke Loader mimics legitimate browsing via bing.com, microsoft.com.
T Tampering Flash exploit corrupts ByteArray memory via CVE-2015-8651. bprocess.exe modifies system to install XMRig miner.
R Repudiation Encrypted 63-byte C2 payloads. Bare IP for malware download (no domain trail). Monero for untraceable payment.
I Info Disclosure XMRig sends OS details to mining pool. Smoke Loader beacons system information to C2 via HTTP POST.
D Denial of Service CPU consumed by XMRig = effective DoS on workstation. Directly explains Charlie's slow performance symptoms.
E Elevation of Privilege Flash exploit gains arbitrary code execution from browser context. No user interaction required beyond browsing.

CIA Triad — Impact Assessment

I Integrity HIGH Flash exploit corrupts ByteArray memory. bprocess.exe (NSIS dropper) modifies system and installs XMRig as unauthorised miner.
A Availability MEDIUM XMRig CPU hijack = effective DoS on workstation. Directly caused Charlie's slow-performance helpdesk ticket.

15 Conclusion

Conclusion

  • This attack demonstrates a fully automated exploitation chain - no user interaction required beyond browsing
  • The outdated Flash Player was the single point of failure that enabled the entire attack
  • All three phases were detected by the IDS - but detection without response allows the attack to succeed
  • The 12-minute window from exploitation to active mining shows the speed of modern automated attacks
  • Defence in depth is essential: patching, monitoring, endpoint protection, and user awareness must work together

Key Lesson: Keeping software up to date is the most cost-effective security control available.

The attacker's Monero wallet provides a potential attribution lead for law enforcement.
